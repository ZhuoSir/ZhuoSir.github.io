HashMap和HashTable的对比

2017-03-21

HashMap

HashMap是实现了Map接口的实现类，是将键映射到值得对象，其中键和值都是对象，并且不能包含重复键（key），但是可以包含重复值（value）。HashMap允许null key 和 null value的存在，但是HashTable是不允许的，并且HashMap不是线程安全的，在多线程环境中，需要实现外同步（Synchronize）；其实HashMap属于HashTable的轻量级实现；因为是非线程安全，所以效率上要高于HashTable；

HashTable

HashTable不仅实现了Map接口，而且继承了Dictionary类，不允许出现key, value 为null的情况，并且属于线程安全的，不用在程序外部实现外同步，HashTable所有的方法都是同步的（Synchronize）；在方法上，HashTable存在contains方法，但是HashMap有containsKey,containsValue两方法；

效率

在效率上方面，两个类都是hash/rehash算法，所以差别不大，只是HashMap是非线程安全，所以效率可能略高；
